2. SMC拿掉流动性后的止损区，会成为新的诱多诱空位置吗？

   > > 聪明钱“是否继续扫”，取决于以下条件：
   >
   > 1. **是否完成了建仓/出货目的？**
   >    - 如果第一轮扫单已经实现了仓位计划，那就不会再扫
   > 2. **是否触发了新结构？**
   >    - 如果扫完之后价格快速收回，并出现结构反包 → 代表方向明确，没必要继续搅动
   > 3. **是否还有大量挂单没成交？**
   >    - 如果流动性没扫干净、市场仍犹豫，那有可能二次扫
   >
   > > 如何应对这种“二次扫”的风险？
   >
   > 1. 止损设在结构之后的更外侧，而不是刚好放在K线高低点（加 buffer）
   > 2. 等信号K线+确认结构（如：二次不破 + 强阳 +前高/低未破）
   > 3. 不要在第一次触碰就激进入场，等市场“犯错”再出手

   > > **问题**：F5经常被骗线 → 信号太多、有效性差、止损频繁

   > * 本质上是把系统从“技术驱动”往“结构驱动”收束，是进阶而非退步
   >
   >   |              | 原系统               | 修改后              |
   >   | ------------ | -------------------- | ------------------- |

   >   | 止损触发概率 | 高频打脸             | 触发更少但更重      |
   >   | 信号过滤     | 依赖结构判断+快K组合 | 本身自带过滤机制    |

   >

   >   | 缺点     | 极易被扫、真假难分       | 入场略滞后，部分空间放弃 |

   >    3. 原理：价格未曾染指，会吸引价格回补
   > 2. iFVG：原FVG被1根或多根强趋势K穿越，原FVG改名叫iFVG
   >    1. 作用：拒绝者，拒绝价格穿越，即支撑/阻力作用
   >    2. 原理：曾经的处女地已经被染指，会阻止其他人进入
   > 3. 用法
   >    1. 只关注折价区的FVG

3. 笔记梳理

   > > 核心理念：摒弃趋势思维，市场中没有“趋势”，只有“结构”
   >
   > - 内部结构 vs 外部结构
   > 
   >   ![image-20250713093839764](https://raw.githubusercontent.com/jiangsai0502/PicBedRepo/master/image-20250713093839764.png)
   >
   >   ![image-20250713093854378](https://raw.githubusercontent.com/jiangsai0502/PicBedRepo/master/image-20250713093854378.png)
   >
   > - 单腿运动：一个没有有效回撤的波段，即为一个单腿运动，即连续上涨、连续下跌
   > 
   > - 有效回撤：例如上涨波段中，前1根K线的低点被下破（实体/影线下破都行），称为有效回撤
   > 
   >  - 即这个有效回撤终结了前一个上涨是波段
   > 
   > - 描述结构的步骤（下跌波段为例）
   > 
   >  1. 下跌波段中，从起点A开始，找高点比前1根高点更高的K线C，找到AC之间的最低点B，连接A的高点和B的低点
   > 
   >  2. 回调的上涨波段中，从起点A开始，找低点比前1根低点更低的K线C，找到AC之间的最高点B，连接A的低点和B的高点
   > 
   >   3. 原则：剔除内包行为（含K线内包、波段内包）
   > 
   >     > 回调段是推动段的内包，但推动段不能是回调段的内包
   >

3. 笔记

   > * 流动性：是散户止损单聚集的区域，通常位于前高/前低等关键点位
   >
   > * 流动性清扫：大资金通过清扫散户止损单来收集筹码，形成OB（建仓区）和FVG（未填补区）
   >
   > * 流动性与OB/FVG的关系
   >
   >   * 流动性清扫是前提：清扫流动性后才能形成有效OB/FVG
   >   * OB/FVG依赖流动性：OB是流动性清扫后的建仓区域，FVG是清扫后留下的未填补区间
   >
   > * 内部流动性：指**波段高低点之内**的流动性
   >
   > * 外部流动性：指**波段高低点之外**的流动性
   >
   > * 流动性、FVG、OB配合使用
   >
   >   | 策略类型             | 入场点                         | 止损位置 | 止盈位置                |
   >   | -------------------- | ------------------------------ | -------- | ----------------------- |
   >   | **高概率，低盈亏比** | FVG下沿/OB顶部，确认流动性清扫 | OB底部   | FVG上沿，或下一流动性点 |
   >   | **低概率，高盈亏比** | OB中轨，确认流动性清扫         | OB底部   | FVG上沿                 |
   >
   > * 假破的方向，散户清扫流行性是吸筹；实体突破的方向，散户流动性是燃料

     - 散户：误以为上涨只是前一波下跌的“回调”，他们会在前高挂空单
     - 主力：顺势打压价格造成**空头假象**，吸引散户建空单仓位
     - 最后再次拉升价格，利用散户的空单止损“推升价格”
   - OB之后如果没有FVG，说明可能并非机构主导

   1. 结构高点：前高处为潜在阻力，存在获利盘出逃动机；等高点附近聚集止损，利于清扫前止盈
   2. 下一个FVG：下轨为最早接触区域，价格触达概率最高；越往上概率越低
2. **止盈完不要继续幻想还会涨**，这不是你的交易了；
3. 如果后续还上涨 → 那是**下一笔交易的计划**；

   >

   >    * sell stop：挂单卖出（止损卖出）

   >
   >    > 用户认为未来的行情将会先上涨后下跌

   >
   > 4. sell stop limit：卖出止损限价
   >
   >    > 用户认为未来的行情将会先下跌后上涨
   >
   >    1. 设置 “价位-  提醒价” 来确认下跌
   >    2. 设置 “停损限价 - 成交价” 来确认上涨
   >    3. “价位-  提醒价”  必须小于 “停损限价 - 成交价”

   void DrawOrUpdateLine(const string name, const double price, const color clr, const int width)
   {
      if(price <= 0.0) return;
      long cid = 0;
      int handle = ObjectFind(cid, name);
      if(handle == -1)
      {
         ObjectCreate(cid, name, OBJ_HLINE, 0, 0, price);
         ObjectSetInteger(cid, name, OBJPROP_SELECTABLE, true);
         ObjectSetInteger(cid, name, OBJPROP_BACK, false);
         ObjectSetInteger(cid, name, OBJPROP_STYLE, STYLE_SOLID);
         ObjectSetInteger(cid, name, OBJPROP_WIDTH, width);
         ObjectSetInteger(cid, name, OBJPROP_COLOR, (long)clr);
      }
      else
      {
         double cur_price = ObjectGetDouble(cid, name, OBJPROP_PRICE);
         long cur_width   = ObjectGetInteger(cid, name, OBJPROP_WIDTH);
         long cur_color   = ObjectGetInteger(cid, name, OBJPROP_COLOR);
   
         if(MathAbs(cur_price - price) > SymbolInfoDouble(_Symbol,SYMBOL_POINT)/10.0)
            ObjectSetDouble(cid, name, OBJPROP_PRICE, price);
   
         if(cur_width != width)
            ObjectSetInteger(cid, name, OBJPROP_WIDTH, width);
   
         if(cur_color != (long)clr)
            ObjectSetInteger(cid, name, OBJPROP_COLOR, (long)clr);
      }
   }
   
   void DeleteIfExists(const string name)
   {
      if(ObjectFind(0,name) != -1)
         ObjectDelete(0,name);
   }
   
   bool PositionExists(ulong ticket)
   {
      int total = PositionsTotal();
      for(int i=0; i<total; i++)
      {
         if(!pos.SelectByIndex(i)) continue;
         if((ulong)pos.Ticket() == ticket) return true;
      }
      return false;

   void CleanupStaleObjects()
   {
      int count = ObjectsTotal(0,-1,-1);
      for(int i=count-1;i>=0;i--)
      {
         string name = ObjectName(0,i);
         if(StringFind(name,Prefix,0)!=0) continue;
   
         int first  = StringFind(name,"_",0);
         int second = StringFind(name,"_",first+1);
         if(first<0 || second<0) continue;
   
         string ticket_str = StringSubstr(name,second+1);
         ulong ticket = (ulong)StringToInteger(ticket_str);
   
         if(!PositionExists(ticket))
            ObjectDelete(0,name);
      }

   // ================= SL/TP 计算函数 =================
   void CalculateSLTP(double open_price,int direction,double &sl,double &tp)
   {
      double point = SymbolInfoDouble(_Symbol,SYMBOL_POINT);
      if(!UseProportion)
      {
         sl = open_price - direction * StopLossPoints * point;
         tp = open_price + direction * TakeProfitPoints * point;
      }
      else
      {
         double atr = iATR(_Symbol, PERIOD_CURRENT, 14);
         if(atr <=0) { sl=tp=0; return; }
         sl = open_price - direction * atr * SLProportion;
         tp = open_price + direction * atr * TPProportion;
      }
      sl = NormalizeDouble(sl,_Digits);
      tp = NormalizeDouble(tp,_Digits);
   }
   
   // ================= 挂单SL/TP函数 =================
   bool SetSLTPForPending(ulong ticket)
   {
      if(!OrderSelect(ticket)) { Print("挂单选择失败: ",ticket); return false; }
      long type = OrderGetInteger(ORDER_TYPE);
      if(type<ORDER_TYPE_BUY_LIMIT || type>ORDER_TYPE_SELL_STOP) return false;

      req.action = TRADE_ACTION_MODIFY;
      req.order  = ticket;
      req.symbol = _Symbol;
      req.price  = open_price;
      req.sl     = sl;
      req.tp     = tp;

         Print("修改挂单失败: ",ticket,", 错误:",GetLastError(),", 返回码:",res.retcode);
      else
         Print("挂单SL/TP设置成功: ",ticket," SL=",sl," TP=",tp);
   
      // 画线
      DrawOrUpdateLine(Prefix+"Open_"+ticket,open_price,OpenColor,LineWidth);
      DrawOrUpdateLine(Prefix+"SL_"+ticket,sl,SLColor,LineWidth);
      DrawOrUpdateLine(Prefix+"TP_"+ticket,tp,TPColor,LineWidth);
   
      return true;
   }

   bool SetSLTPForPosition(ulong ticket)
   {

      string symbol = PositionGetString(POSITION_SYMBOL);
      if(symbol!=_Symbol) return false;

      DrawOrUpdateLine(Prefix+"SL_"+ticket,sl,SLColor,LineWidth);
      DrawOrUpdateLine(Prefix+"TP_"+ticket,tp,TPColor,LineWidth);

   }
   
   // ================= 动态更新线条 =================
   void UpdateLinesFollow()
   {
      int total = PositionsTotal();
      for(int i=0;i<total;i++)
      {
         if(!pos.SelectByIndex(i)) continue;
         if(pos.Symbol()!=_Symbol) continue;
   
         ulong ticket = (ulong)pos.Ticket();
         double open_price = pos.PriceOpen();
         double sl = pos.StopLoss();
         double tp = pos.TakeProfit();
   
         DrawOrUpdateLine(Prefix+"Open_"+ticket, open_price, OpenColor, LineWidth);
         if(sl>0) DrawOrUpdateLine(Prefix+"SL_"+ticket, sl, SLColor, LineWidth);
         else DeleteIfExists(Prefix+"SL_"+ticket);
         if(tp>0) DrawOrUpdateLine(Prefix+"TP_"+ticket, tp, TPColor, LineWidth);
         else DeleteIfExists(Prefix+"TP_"+ticket);
      }
   
      int total_orders = OrdersTotal();
      for(int i=0;i<total_orders;i++)
      {
         ulong ticket = OrderGetTicket(i);
         if(OrderSelect(ticket))
         {
            long type = OrderGetInteger(ORDER_TYPE);
            if(type>=ORDER_TYPE_BUY_LIMIT && type<=ORDER_TYPE_SELL_STOP)
            {
               double open_price = OrderGetDouble(ORDER_PRICE_OPEN);
               double sl = OrderGetDouble(ORDER_SL);
               double tp = OrderGetDouble(ORDER_TP);
   
               DrawOrUpdateLine(Prefix+"Open_"+ticket, open_price, OpenColor, LineWidth);
               if(sl>0) DrawOrUpdateLine(Prefix+"SL_"+ticket, sl, SLColor, LineWidth);
               else DeleteIfExists(Prefix+"SL_"+ticket);
               if(tp>0) DrawOrUpdateLine(Prefix+"TP_"+ticket, tp, TPColor, LineWidth);
               else DeleteIfExists(Prefix+"TP_"+ticket);
            }
         }
      }
   }

   void OnTimer()
   {
      static uint last_cleanup = 0;
      uint now = GetTickCount();

1. [价格传递，揭秘价格波动的背后逻辑](https://www.bilibili.com/video/BV1ngrMYeEaA/?spm_id_from=333.1387.collection.video_card.click)

   > - **价格传递**：ICT最核心的内功心法，是ICT交易的“道”层面指导
   >
   > - **OB**：机构建仓区，是上涨前的一根或多根阴线，OB是强支撑位，回调不会轻易跌破
   > - **FVG**：快速上涨导致的K线缺口，反映买方力量远超卖方，回调时，低价买家重新入场，推高价格
   > - **流动性**：机构利用回调来清扫底部流动性，完成筹码买入，利用突破来清扫顶部流动性，推高价格
   > - **价格传递的四个阶段**
   >
   >   - **积累**：机构在低位积累筹码，可能通过操纵来清扫底部流动性以获取更多筹码
   >   - **扩张**：拉升价格突破关键位，触发顶部流动性，助力推高价格
   >   - **回调**：回调至内部流动性区域（如FVG或OB），触发未成交的买单，再次助力推高价格
   >
   >   - **反转**：达到机构目标后，抛售导致价格反转，完成本次价格传递
   >
   > - **价格传递的核心逻辑**：机构通过获取外部流动性、回补内部流动性、平衡FVG来完成价格传递
   >
   > - **回调的3大原因**：通过回调构造形态、FVG，**① 构造并清扫内部流动性**，进而推动下跌，再高卖低买 **② 降低持仓成本**，最终 **③ 降低持仓风险**，避免更大资金介入压低价格导致亏损

   >
   > - **多周期分析的核心步骤**
   >   1. **划分结构**：识别当前及以上级别的结构，明确价格传递形式
   >   1. **观察POI是否生效**：观察关键位（如OB、FVG、OTE）是否触发反应
   >   1. **观察次级别是否背驰**：通过次级别背驰判断本级别回调是否结束
   > - **步骤详解**
   >   - **划分结构**：单一周期无法确认是否回调到位，结合大级别POI，才能明确价格传递的终点
   >     - 例：H1 空转空（向下传递），但需确认H4是否到达POI，进而判断是否回调到位
   >   - **判断POI是否生效**
   >     - **POI**：OB、FVG、OTE，是价格传递的潜在目标或支撑/阻力
   >     - **生效**：① **POI级别**出**信号K + 高1/低1** ② **信号K级别**出**结构反转**   
   >     - **例**：H4的POI上，H4出影线 + 信号K + 高1，则POI生效，价格可能反转；若POI未生效，仅触发小幅反弹，需看下一步：观察次级别背驰
   >
   >   - **观察次级别是否背驰**
   >     - 若H1下跌背驰，则H4回调大概率结束，目标为H4高点
   >     - 若H1下跌无背驰，回调未结束，则可能仅到H1的POI，需严格管理止盈

   >   - 低概率位需结合背驰确认

##### 9.22

1. 做单顺序再梳理

   ![image-20250922133404584](https://raw.githubusercontent.com/jiangsai0502/PicBedRepo/master/image-20250922133404584.png)

   1. 外部波段取出IDM之前：做外部波段QML，直到打损（打损即取出IDM）
   2. 外部波段取出IDM之后
      1. 吃结构回调：按回调方向做回调外部波段QML，直到决策POI
      2. 在决策POI做choch模型：等决策POI出反转信号（如本级别IFC（次级别QML），本级别QML）
         1. 有反转信号：做结构顺势
         2. 无反转信号：继续吃结构回调到极端POI
      3. 在极端POI做choch模型：等极端POI出反转信号（如本级别IFC（次级别QML），本级别QML）
         1. 有反转信号：做结构顺势
         2. 无反转信号：结构被反转
